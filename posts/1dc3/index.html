DolphinDB编程语言入门 + 用户手册笔记

DolphinDB语言特点：
多范式，即程序范式有不同写法：命令式、向量式、函数式、SQL范式

DolphinDB 是列式数据库系统，在内存中以列为单位连续储存数据。DolphinDB 从左向右以列为单位填充矩阵中的元素。

强类型：
一般的标量和向量，都有明确的数据类型。不推荐使用元组(Tuple)

处理空值的一般规则：四则运算时，若有空值参与运算，则结果为空；聚合计算、窗口计算时，忽略空值。其它计算，遇到空值，结果为空值。
大小比较比较特殊，默认是把空值作为最小值来处理。

技术分析（Technical Analysis）指标库

ta是Python库TA-Lib的DolphinDB实现

例子

use ta    
close = 7.2 6.97 7.08 6.74 6.49 5.9 6.26 5.9 5.35 5.63
x = wma(close, 5)   // weighted moving average
print x;


close = 7.2 6.97 7.08 6.74 6.49 5.9 6.26 5.9 5.35 5.63 3.81 3.935 4.04 3.74 3.7 3.33 3.64 3.31 2.69 2.72
date = (2020.03.02 + 0..4 join 7..11).take(20)
symbol = take(`F,10) join take(`GPRO,10)
t = table(symbol, date, close)
update t set wma = wma(close, 5) context by symbol;  // context by股票代码，计算5天加权移动平均，更新表格

驼峰命名法

ta 模块中，函数名及参数名均采用驼峰式命名法。

指混合使用大小写字母来构成变量和函数的名字。当变量名或函数名是由一个或多个单词连结在一起构成的唯一识别字时，第一个单词以小写字母开始，从第二个单词开始以后的每个单词的首字母都采用大写字母，例如：myName、myAge，这样的变量名看上去就像骆驼峰一样此起彼伏，因此被称为驼峰命名法。

bBands函数 计算布林线指标

upperband, middleband, lowerband = BBANDS(close, timeperiod=5, nbdevup=2, nbdevdn=2, matype=0)

close：收盘价
timeperiod:计算周期，一般选择20天
nbdevup:上轨线标准差的倍数
nbdevdn:下轨线标准差的倍数
matype:移动平均类别，0代表简单移动平均。移动平均线又可分为简单移动平均数（MA）、加权移动平均数（WMA，赋予不同日期不同的加权份量）和指数平滑移动平均线数（EMA，平滑因子小时较圆滑，平滑因子大时较为尖锐）三种

参考连接

select *, bBands(close, 5, 2, 2, 2) as `high`mid`low from t context by symbol

几个join

// left join/left outer join
// lj或者lsj(leftTable, rightTable, matchingCols, [rightMatchingCols], [leftFilter], [rightFilter])

t1 = table(5 3 1 as id,  300 500 800 as qty);
t2 = table(1 2 3 3 as id, 7.8 4.6 5.1 0.1 as value)
lj(t1, t2, `id)
lsj(t1, t2, `id)
// 左连接(lj)返回左表中所有与右表匹配的记录。如果右表中没有匹配的记录，将会返回NULL。如果右表中有多条匹配记录，将会返回所有的匹配记录。因此，lj返回结果的行数有可能比左表的行数多。
// 左半连接(lsj)和左连接(lj)的唯一区别是，如果右表中有多条匹配记录，lsj将会取第一条的匹配记录。因此，lsj返回结果的行数与左表的行数相等。


// asof join
// 常用于匹配时间，类似于lsj，如果匹配不到，选最近的时间进行匹配。（类似于模糊匹配）
// 注意“假定右表已按照连接列排过序”，即函数的使用条件是右表经过排序
t1 = table(2015.01.01+(0 31 59 90 120) as date, 1.2 7.8 4.6 5.1 9.5 as value);t1;
t2 = table(2015.02.01+(0 15 89 89) as date, 1..4 as qty);t2;
select * from lsj(t1, t2, `date);
select * from aj(t1, t2, `date);

// t2没有按日期顺序，连接的话，结果会产生错误，0301 0401明明和0216更近，却匹配了20150216
t2 = table(2015.02.01+(0 15 -89 89) as date, 1..4 as qty);t2;
select * from lsj(t1, t2, `date);
select * from aj(t1, t2, `date);


// window join   感觉这个函数很有用，有空细看！！！
// wj(leftTable, rightTable, window, aggs, matchingCols, [rightMatchingCols])
// aggs是聚合函数
t1 = table(`A`A`B as sym, 09:56:06 09:56:07 09:56:06 as time, 10.6 10.7 20.6 as price)
t2 = table(   
        take(`A, 10) join take(`B, 10) as sym, 
        take(09:56:00+1..10, 20) as time, 
        (10 + (1..10) \ 10 - 0.05) join (20 + (1..10) \ 10 - 0.05) as bid, 
        (10 + (1..10) \ 10 + 0.05) join (20 + (1..10) \ 10 + 0.05) as offer, 
        take(100 300 800 200 600, 20) as volume
      )   //换行的话 要全部选中一起运行
      
wj(t1, t2, -5:0, , `sym`time);

t3 = t2
t3.rename!(`time, `second) // 把time rename成second
wj(t1, t3, -2:2, <[wavg(bid,volume), wavg(offer,volume)]>, `sym`time, `sym`second);
